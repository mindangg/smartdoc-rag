Bối cảnh và động lực
Với khối lượng thông tin ngày càng tăng trong các tài liệu số, việc tìm kiếm và trích xuất
thông tin một cách hiệu quảđã trởthành thách thức lớn. Các phương pháp truyền thống
như tìm kiếm từkhóa thường không đủđểhiểu ngữcảnh và ý nghĩa sâu xa của câu hỏi.
Hệthống RAG (Retrieval-Augmented Generation) kết hợp ưu điểm của:
• Information Retrieval: Tìm kiếm thông tin liên quan từcơ sởdữliệu
• Natural Language Generation: Sinh ra câu trảlời tựnhiên dựa trên thông tin
đã tìm được
1.2
Mục tiêu dựán
Dựán đặt ra các mục tiêu cụthểsau:
1. Xây dựng giao diện web thân thiện cho phép người dùng tải lên tài liệu PDF
2. Tích hợp công nghệembedding đểchuyển đổi văn bản thành vector
3. Triển khai vector database (FAISS) đểlưu trữvà tìm kiếm hiệu quả
4. Tích hợp mô hình ngôn ngữlớn Qwen2.5:7b được tối ưu cho tiếng Việt
5. Xây dựng pipeline RAG hoàn chỉnh từdocument loading đến answer generation
6. Tối ưu hóa hiệu suất và độchính xác của câu trảlời
1.3
Phạm vi dựán
Dựán tập trung vào:
• Xửlý tài liệu PDF đa ngôn ngữ
• Hỗtrợxuất sắc cho tiếng Việt và 50+ ngôn ngữkhác
• Tựđộng phát hiện ngôn ngữvà trảlời phù hợp
• Chạy local trên máy tính cá nhân
• Sửdụng mô hình open-source và miễn phí

CƠ SỞLÝ THUYẾT
2.1
RAG (Retrieval-Augmented Generation)
2.1.1
Khái niệm
RAG là một kỹthuật trong NLP kết hợp hai thành phần:
1. Retrieval (Truy xuất): Tìm kiếm thông tin liên quan từcơ sởdữliệu
2. Generation (Sinh): Sửdụng LLM đểsinh ra câu trảlời dựa trên thông tin đã
truy xuất
2.1.2
Kiến trúc RAG
Kiến trúc RAG cơ bản bao gồm:
RAG(x) = Generator(x, Retriever(x))
(1)
Trong đó:
• x: Câu hỏi đầu vào
• Retriever(x): Hàm truy xuất các tài liệu liên quan
• Generator: Mô hình sinh câu trảlời
2.2
Embedding và Vector Database
2.2.1
Text Embedding
Text embedding là quá trình chuyển đổi văn bản thành vector sốcó sốchiều cốđịnh:
E : Text →Rd
(2)
Trong đó d là sốchiều của vector (thường là 384, 512, 768, v.v.)
Đặc điểm quan trọng:
• Văn bản có nghĩa tương tựsẽcó vector gần nhau trong không gian d chiều
• Khoảng cách được tính bằng cosine similarity hoặc Euclidean distance
2.2.2
FAISS (Facebook AI Similarity Search)
FAISS là thư viện hiệu quảcho similarity search và clustering của dense vectors:
Ưu điểm:
• Tìm kiếm nhanh với hàng triệu vector
• HỗtrợGPU acceleration
• Memory-efficient indexing

2.3
Large Language Models (LLMs)
2.3.1
Qwen2.5
Qwen2.5 là mô hình ngôn ngữlớn phát triển bởi Alibaba Cloud:
Đặc điểm:
• Kiến trúc transformer-based thếhệmới
• Hỗtrợcontext dài lên đến 128K tokens
• Tối ưu xuất sắc cho tiếng Việt và đa ngôn ngữ
• Phiên bản 7B parameters cân bằng giữa hiệu suất và chất lượng
• Được train trên corpus đa ngôn ngữbao gồm tiếng Việt
• Performance vượt trội so với Llama3 cho tiếng Việt
2.3.2
Ollama
Ollama là framework đểchạy LLMs locally:
Tính năng:
• API đơn giản, dễsửdụng
• Hỗtrợnhiều mô hình khác nhau
• Tối ưu performance cho local machine
• Không cần internet sau khi tải model
2.4
LangChain Framework
LangChain là framework Python đểxây dựng ứng dụng với LLMs:
Thành phần chính:
• Document Loaders: Tải và parse các định dạng tài liệu
• Text Splitters: Chia văn bản thành chunks nhỏ
• Embeddings: Chuyển đổi text thành vectors
• Vector Stores: Lưu trữvà tìm kiếm vectors
• Retrievers: Truy xuất thông tin liên quan
• Chains: Kết nối các thành phần lại với nhau

3.1
Kiến trúc tổng quan
Hệthống được thiết kếtheo mô hình multi-layer architecture:
Presentation Layer (Streamlit)
• File Upload Interface
• Question Input Interface
• Answer Display
Application Layer (LangChain)
• Document Processing Pipeline
• RAG Chain Management
• Prompt Engineering
Data Layer
• FAISS Vector Store
• Multilingual MPNet Embeddings (768-dimensional)
• PDF Document Storage
Model Layer (Ollama)
• Qwen2.5:7b Model
• LLM Inference Engine
Hình 1: Kiến trúc hệthống RAG
3.2
Data Flow (Luồng dữliệu)
3.2.1
Document Processing Flow
1. Upload: Người dùng upload PDF file
2. Loading: PDFPlumberLoader đọc nội dung PDF
3. Splitting: RecursiveCharacterTextSplitter chia thành chunks
4. Embedding: Multilingual MPNet (768-dim) chuyển chunks thành vectors
5. Indexing: FAISS lưu trữvectors vào index

Query Processing Flow
1. Query Input: Người dùng nhập câu hỏi
2. Query Embedding: Chuyển câu hỏi thành vector

4. Context Building: Kết hợp các chunks thành context
5. Prompt Construction: Tạo prompt với context và question
6. LLM Inference: Qwen2.5:7b sinh câu trảlời
